# OnionReel Roadmap

## Phase 0 — Align (today)
- Define success criteria: what “monster” means in measurable terms.
  - Craig’s priority: **ALL OF IT** (speed + cinematic quality + repeatability + automation).
- Pick initial target workflow to perfect (recommendation below).

### Recommendation: flagship workflow
- Start with **30–60s paid ad** pipeline (most reusable across niches/platforms):
  brief → concept → script → storyboard → shot list → asset plan → edit plan → cut → sound → color → captions → exports + QC.

## Phase 1 — Foundation (Brain + KB + Quality System skeleton)
- Workspace structure + naming conventions
- Knowledge taxonomy + ingestion rules
- Prompt library v1 (role-based)
- **Quality System v1**: rubrics + acceptance criteria per deliverable
- Production lifecycle checklists (pre/pro/post)

## Phase 2 — Workflow Engine (the “perfected pipeline”)
- Canonical pipeline for 30–60s ads (brief → final)
- Variant engine (hooks/CTAs/lengths/aspect ratios)
- Versioning rules (naming, approvals, change logs)

## Phase 3 — Dashboard v1 (local web app)
- Project + asset tracker
- Brief/script/storyboard/shot-list modules
- Review + versioning + notes
- QC reports view (pass/fail + fixes)

## Phase 4 — Brain v1 (orchestration)
- Job runner (tasks, retries, logs)
- Artifact store (scripts/boards/exports/versions)
- Integrations (Drive/Frame.io/Dropbox, Telegram, NLE exports)

## Phase 5 — Automation + QC moat (hardening)
- Automated QC (audio loudness, peaks, captions, safe margins, export specs)
- Style consistency checks (color/audio/pacing heuristics)
- Benchmark library (best-in-class ads by niche)

## Phase 6 — Scale
- Multi-project scale + templates
- Style packs per brand/client
- Permissions + analytics + billing hooks
